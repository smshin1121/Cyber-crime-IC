# Codesight Codemap — Cyber_crime_IC

_Generated by `tools/codesight_scan.py` using [smshin1121/codesight](https://github.com/smshin1121/codesight) v1.10.0._

This directory contains AI context maps for the **code modules** of this wiki
project (`tools/` Python utilities, `web/` Flask app). The wiki content
itself (`wiki/`, `raw/`, `docs/`) is intentionally excluded — codesight maps
*code*, not knowledge content.

## Why this exists

The wiki project mixes two very different layers:

| Layer | Directories | Tooling for understanding |
|---|---|---|
| **Code** | `tools/`, `web/` | codesight (this directory) |
| **Knowledge content** | `wiki/`, `raw/`, `docs/` | Obsidian wikilinks, `index.md`, `tools/lint.py` |

When working with AI assistants (Claude Code, Cursor) on code changes, this
directory gives the assistant a **compact map of the code surface** so it does
not waste tokens crawling 460+ markdown content files looking for Python.

## How to regenerate

```bash
python tools/codesight_scan.py --clean
```

Or with HTML report:

```bash
python tools/codesight_scan.py --clean --html
```

The helper script wraps `npx codesight` and works around two issues in
codesight 1.10:
1. The `-o` flag interprets absolute paths relative to the scanned directory,
   producing nested duplicate paths. The helper writes to `<src>/.codesight/`
   then moves the output to `.codesight/<module>/`.
2. `.codesightignore` patterns are not respected when scanning the project
   root, so each code module is scanned individually.

## When to regenerate

- After significant changes to `tools/*.py` or `web/*.py`
- Before starting a complex code-refactoring conversation with an AI assistant
- As part of the build pipeline (optional — not currently automated)

## Limitations

- codesight does not understand Markdown wiki content. The wiki schema lives
  in [`CLAUDE.md`](../CLAUDE.md) (project root). Use `tools/lint.py` to check
  wiki health.
- codesight's "middleware" detector mistakes wiki frontmatter `validation:`
  fields for HTTP middleware when the wiki is not excluded — this is why we
  scan code modules in isolation.
- Not connected to the auto-collection pipeline (`tools/auto_collect.py`).
  This is purely a code-side dev tool.
